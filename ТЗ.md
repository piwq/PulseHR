## **Цифровой сервис «PulseHR»** 

## **Цель проекта** 

Разработать корпоративный сервис опросов сотрудников, позволяющий HR-службе и руководителям получать регулярную обратную связь от сотрудников, анализировать вовлечённость и оперативно выявлять проблемные зоны внутри компании. 

Система должна заменить использование Google Forms и иных внешних сервисов, обеспечив централизованное хранение данных, гибкую аналитику и поддержку анонимных опросов. 

## **Бизнес-проблема** 

- В настоящее время опросы сотрудников проводятся через сторонние 

- сервисы и Excel, что создаёт ряд ограничений: 

   - отсутствует единая система хранения результатов; 

   - сложно анализировать данные по подразделениям и периодам; 

   - сотрудники сомневаются в анонимности опросов; 

   - нет автоматического расчёта eNPS и других HR-метрик; 

   - HR-специалисты тратят много времени на ручную обработку результатов; 

   - отсутствует инструмент регулярного pulse-мониторинга настроения 

сотрудников; 

- сотрудники не узнают о появлении новых опросов вовремя – рассылка ссылок вручную через мессенджеры и e-mail приводит к низкому охвату и задержкам прохождения. 

- В результате компания получает неполную и несвоевременную обратную 

- связь, а управленческие решения принимаются без достаточной аналитики. **Целевая аудитория** 

## **Основные пользователи** 

- HR-специалисты; 

- Директор по персоналу; 

- руководители подразделений; 

- сотрудники компании. 

## **Основной функционал MVP** 

## **Авторизация сотрудников** 

Сотрудник проходит авторизацию по номеру телефона с подтверждением через OTP-код (SMS или имитация в MVP). 

- Один номер телефона = один аккаунт сотрудника. Система должна: 

- исключать повторное прохождение одного и того же опроса; 

- сохранять авторизованную сессию; 

- поддерживать разделение ролей: 

- 1) HR / Администратор; 

- 2) Сотрудник. 

## **Конструктор опросов** 

HR-специалист должен иметь возможность создавать опросы через вебинтерфейс. 

Поддерживаемые типы вопросов: 

- одиночный выбор; 

- множественный выбор; 

- шкала оценки (NPS / eNPS); 

- текстовый ответ; 

- матричные вопросы. 

Настройки опроса: 

- название и описание; 

- сроки проведения; 

- режим анонимности; 

- выбор целевой аудитории (роли); 

- статусы: 

- 1) черновик; 

- 2) активный; 

- 3) завершён; 

- 4) архив. 

## **Ветвление сценариев опроса** 

– Система должна поддерживать conditional logic отображение разных 

вопросов в зависимости от ответов сотрудника. 

Пример: 

Вопрос 1: «Выберите вашу должность» 

   - если «Товаровед» – показать вопросы блока А (про работу на точке); 

- если «Руководитель» – показать вопросы блока Б (про управление 

- командой); 

- если «Офисный сотрудник» – показать вопросы блока В (про офисную 

- среду). 

Вопрос 2 (блок А): «Оцените удобство рабочего места» 

- если оценка 1-2 – показать уточняющий вопрос «Что именно неудобно?»; 

- если оценка 3-5 – перейти к следующему вопросу. 

## **Режимы конфиденциальности** 

Система должна поддерживать два режима проведения опросов: 

## **Анонимный режим** 

- ответы сотрудников не связаны с личностью; 

- HR не может определить автора ответа; 

- перед прохождением отображается уведомление об анонимности. 

## **Идентифицированный режим** 

- ответы привязаны к сотруднику; 

- HR видит автора ответа и подразделение (роль).

## **Уведомления о новых опросах** 

Для повышения охвата и скорости прохождения опросов система должна автоматически уведомлять сотрудников о появлении новых опросов через несколько каналов с приоритетной доставкой. Реализуется каскадная схема: основной канал – Web Push, резервный – SMS, дополнительные – Telegram-бот и e-mail. 

## **Web Push – основной канал доставки** 

При первом входе сотрудника на веб-портал PulseHR с поддерживаемого браузера (Chrome, Edge, Firefox, Safari, Yandex Browser) система должна отображать запрос разрешения на отправку push-уведомлений. Поведение и техническая реализация: 

- перед нативным запросом браузера показывается мягкое объяснение «Получайте уведомления о новых опросах прямо в браузере – не пропустите ни один опрос» с кнопками «Включить» / «Не сейчас»; 

- при согласии на стороне клиента вызывается Notification.requestPermission() и подписка через Push API (PushManager.subscribe); 

   - регистрируется Service Worker (sw.js) с обработчиком события push; 

   - сервер использует VAPID-ключи и протокол Web Push (RFC 8030); 

- идентификатор подписки (endpoint + keys p256dh, auth) сохраняется на 

- сервере и связывается с user_id; 

- поддержка нескольких устройств у одного сотрудника (рабочий ПК, 

- ноутбук, смартфон в браузере) – независимые подписки; 

- при отзыве подписки браузером или удалении – автоматическая чистка 

- недействительных endpoint-ов на сервере. 

   - Содержание push-уведомления: 

   - заголовок: «Новый опрос: {название_опроса}»; 

   - тело: «До окончания – {срок}. Время прохождения – ~{N} мин.»; 

   - иконка приложения; 

   - кнопка действия «Пройти опрос» – открывает прямую ссылку на опрос; 

   - data-payload содержит survey_id для дедупликации и отметки прочтения. 

## **SMS – резервный канал** 

Если сотрудник: 

   - отказался от push-уведомлений в браузере; 

- не открывал веб-портал более 14 дней (push-подписка считается 

- устаревшей); 

- не имеет активной push-подписки ни на одном устройстве, система автоматически переключается на SMS-доставку через интеграцию с SMSпровайдером (МТС, СМС.ру, SMSC.ru или иной). 

Шаблон SMS: 

_«PulseHR: новый опрос «{название}». Пройти: {короткая_ссылка}. До_ 

## _{дата_окончания}.»_ 

Ограничения SMS-доставки: 

   - не более 1 SMS на один опрос на одного сотрудника; 

   - не более 3 SMS в сутки на одного сотрудника (антиспам); 

- отправка только в рабочие часы по часовому поясу сотрудника (с 09:00 до 

- 18:00); 

   - стоимость отправки логируется и отображается в дашборде HR. 

## **Дополнительные каналы** 

Будет плюсом реализация: 

- Telegram-бот – сотрудник привязывает свой Telegram через QR-код в профиле, бот отправляет уведомление с inline-кнопкой «Пройти опрос»; 

- e-mail – для корпоративной рассылки и опросов, требующих формального подтверждения; 

- виджет в корпоративном портале / Битрикс24 – список доступных опросов 

- с индикатором новизны.

## **Триггеры отправки уведомлений** 

Система отправляет уведомления автоматически при наступлении следующих событий: 

- публикация нового опроса (статус «Активный») – уведомление целевой 

- аудитории; 

   - добавление сотрудника в действующий опрос (расширение аудитории); 

   - напоминание непрошедшим – за 48 часов до окончания опроса; 

   - финальное напоминание – за 24 часа до окончания; 

- запуск pulse-опроса настроения по расписанию (например, каждый 

- понедельник в 10:00 по часовому поясу сотрудника). 

## **Настройки уведомлений сотрудником** 

В личном кабинете сотрудник должен иметь возможность: 

- включить или отключить каждый канал (Web Push / SMS / Telegram / e- 

- mail) независимо; 

- просмотреть список устройств с активными push-подписками и удалить 

- ненужные; 

   - выбрать предпочтительное время уведомлений (утром / днём / вечером); 

- временно поставить «не беспокоить» (отпуск, больничный) на указанный 

- период. 

Полностью отключить уведомления о критичных опросах (выходное интервью, обязательные опросы по 152-ФЗ) сотрудник не может – доставка таких уведомлений гарантируется по каскаду. 

## **Защита от спама и комплаенс** 

- общий лимит уведомлений на сотрудника – не более 5 в сутки по всем 

- каналам; 

- дедупликация – одно и то же уведомление по одному опросу не 

- отправляется повторно по одному каналу; 

- журналирование всех отправок (канал, время, статус доставки, стоимость 

- для SMS) для аудита и расчёта эффективности; 

- соблюдение 152-ФЗ – согласие сотрудника на отправку коммуникаций 

- фиксируется при первом входе; 

   - отправка SMS только в рабочее время по часовому поясу сотрудника; 

- при отзыве согласия сотрудником – немедленное прекращение всех 

- автоматических уведомлений (кроме критичных по регламенту). 

## **Метрики доставки** 

В дашборде HR отображается аналитика по каждому опросу: 

- количество отправленных уведомлений по каналам; 

- доставлено / открыто / переход по ссылке (CTR); 

- сравнение скорости ответа по каналам; 

- среднее время от отправки уведомления до начала прохождения опроса; 

- стоимость SMS-доставки за период. 

## **Аналитика и отчётность** 

HR-специалист должен получать аналитические дашборды с 

отображением: 

- процента прохождения опроса; 

- динамики ответов; 

- результатов по подразделениям; 

- распределения ответов; 

- eNPS; 

- текстовых комментариев сотрудников; 

- график прохождений по дням (когда активнее всего заполняли); 

- эффективность каналов уведомлений (Web Push / SMS / Telegram / e-mail). Дополнительно: 

- фильтрация по подразделениям, должностям, городам; 

- экспорт результатов в Excel / CSV; 

- защита от деанонимизации сотрудников, если был выбран анонимный 

опрос. 

## **Пользовательский сценарий** 

## **Сценарий сотрудника** 

- сотрудник авторизуется по номеру телефона; 

- при первом входе разрешает отправку push-уведомлений в браузере; 

   - получает push-уведомление о новом опросе – кликает «Пройти опрос»; 

   - видит доступные опросы; 

   - проходит опрос; 

   - получает подтверждение об успешной отправке ответов. 

   - **Сценарий HR** 

   - HR создаёт опрос; 

   - настраивает ветвление и режим конфиденциальности; 

   - выбирает целевую аудиторию; 

- публикует опрос – система автоматически отправляет уведомления по 

- каскадной схеме; 

- анализирует результаты через дашборд и отчёты, контролирует 

- эффективность каналов уведомлений. 

   - **Дополнительные возможности (как преимущество решения)** Будет плюсом реализация: 

   - Telegram-бота для прохождения опросов прямо в чате (без перехода на 

сайт); 

   - AI-анализа текстовых ответов (sentiment analysis); 

   - рекомендаций для HR на основании результатов; 

   - pulse-опросов настроения; 

   - геймификации участия сотрудников; 

   - AI-помощника для генерации вопросов; 

- PWA (Progressive Web App) – установка веб-приложения на главный экран 

- смартфона с поддержкой push даже в закрытом браузере; 

   - smart-расписание уведомлений на основе истории активности сотрудника 

- (в какие часы он чаще проходит опросы – в эти и слать). 

   - **Технические требования** 

   - Рекомендуемый стек: 

   - Backend: FastAPI / Django / Node.js; 

   - Frontend: React / Vue; 

   - База данных: PostgreSQL; 

   - REST API; 

   - Docker-развёртывание. 

Для подсистемы уведомлений: 

   - Web Push: библиотека pywebpush (Python) или web-push (Node.js), 

- генерация и хранение VAPID-ключей; 

   - Service Worker на стороне клиента для обработки push-событий; 

   - очередь задач (Celery / RQ + Redis) для асинхронной массовой рассылки; 

- интеграция с SMS-провайдером по HTTP API (МТС, СМС.ру, SMSC.ru 

- или иной); 

   - Telegram Bot API – для канала Telegram; 

   - SMTP-шлюз компании – для e-mail; 

   - rate-limit и анти-спам через Redis-счётчики.

## **Потенциальная ценность для бизнеса** 

Решение позволит создать единый внутренний инструмент обратной связи, повысить качество HR-аналитики и обеспечить руководителей актуальными данными о состоянии команд. 

Подсистема уведомлений с приоритетом Web Push даёт значимое снижение расходов на SMS-рассылки при одновременном росте охвата и скорости прохождения опросов. 

В перспективе сервис может стать частью внутренней HR-экосистемы компании.
